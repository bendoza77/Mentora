## 4. CORE PROBLEM

Georgian students face a **triple failure** in exam preparation:

1. **Personalization Gap**: Generic textbooks and YouTube tutorials cannot adapt to individual weaknesses. Every student gets the same content regardless of what they already know.

2. **Cost Barrier**: Private math tutors charge 50–150 GEL/hour (1–3 USD equivalent for students but significant in local purchasing power). A full academic year of tutoring costs more than many families earn monthly.

3. **Feedback Vacuum**: Students solve problems and receive no diagnostic feedback. They don't know *why* they got something wrong, or *which topics* will cost them points on exam day.

**Result**: 42% of Georgian students fail to achieve their target score on national math exams (source: NAEC data, 2023).

## 5. SOLUTION

Mentora AI provides a **three-layer solution**:

### Layer 1: AI Tutor (Problem Solver)
- Student inputs any math problem (text or photo)
- AI returns step-by-step solution with conceptual explanations
- Hint system: progressive hints prevent over-dependence
- Confidence scoring: AI rates its own certainty (guards against hallucination)

### Layer 2: Adaptive Practice Engine
- Analyzes attempt history to identify weak topics
- Generates personalized problem sets weighted toward weaknesses
- Difficulty auto-scales: success → harder problems, failure → easier scaffolding

### Layer 3: Exam Simulation
- Full-length timed mock exams matching real Georgian ENT format
- Auto-grading with solution explanations for every wrong answer
- Post-exam analytics: topic breakdown, time management analysis, comparison to national averages

### Difficulty Rating Algorithm
```python
def compute_problem_difficulty(problem):
    """
    Score 1-10 based on:
    - Concept depth (1-3): basic formula vs. multi-step reasoning
    - Computation complexity (1-3): simple arithmetic vs. multi-variable
    - Trap potential (1-2): common misconceptions likely
    - Topic inherent difficulty (1-2): ENT historical data
    """
    score = (
        concept_depth_score(problem) * 0.35 +
        computation_score(problem) * 0.30 +
        trap_score(problem) * 0.20 +
        topic_baseline(problem.topic) * 0.15
    )
    return round(score, 1)
```

### Weakness Scoring Formula
```javascript
function computeTopicAccuracy(attempts, decayFactor = 0.85) {
  // Recency-weighted accuracy — recent attempts matter more
  // attempts sorted newest-first
  let weightedCorrect = 0;
  let weightedTotal = 0;

  attempts.forEach((attempt, index) => {
    const weight = Math.pow(decayFactor, index);
    weightedTotal += weight;
    if (attempt.isCorrect) weightedCorrect += weight;
  });

  const accuracy = weightedTotal > 0 ? weightedCorrect / weightedTotal : 0;

  // Confidence penalty for low sample size
  const confidence = Math.min(1, attempts.length / 10);

  return {
    accuracy: accuracy * 100,
    confidence: confidence * 100,
    attempts: attempts.length,
    trend: computeTrend(attempts) // 'improving' | 'declining' | 'stable'
  };
}
```

### Error Correction Loop
```javascript
async function generateAIResponse(problem, userContext) {
  const primaryResponse = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages: buildMessages(problem, userContext),
    temperature: 0.1  // low temperature for math accuracy
  });

  // Verify arithmetic in solution
  const verificationResult = await verifyMathematicalSteps(primaryResponse);

  if (!verificationResult.isValid) {
    // Regenerate with explicit verification instruction
    const correctedResponse = await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [...buildMessages(problem, userContext), {
        role: 'user',
        content: 'Please verify your arithmetic step by step before answering.'
      }],
      temperature: 0.05
    });
    return { response: correctedResponse, wasCorreected: true };
  }

  return { response: primaryResponse, wasCorrected: false };
}
```

### AI Cost Optimization
1. **Caching**: Cache solutions for identical problems (MD5 hash of problem text)
2. **Tier routing**: Free users → GPT-3.5-turbo, Paid → GPT-4o
3. **Context compression**: Summarize old session messages instead of sending full history
4. **Problem bank**: Pre-generated solutions for common ENT problems (no API call needed)
5. **Streaming**: Stream responses to reduce perceived latency without extra cost
6. **Token budget**: Max 800 input + 1200 output tokens per request
7. **Estimated cost**: $0.004/problem (GPT-4o) → $0.40/100 problems → well within 15 GEL Pro margin
